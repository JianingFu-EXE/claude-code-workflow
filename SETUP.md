# Cross-Platform Claude Code Config Sync -- Mac Setup

Run these commands **once** on the Mac after OneDrive has synced the `.claude-shared/` folder.

## Prerequisites

- OneDrive is signed in and syncing `Projects/.claude-shared/`
- `~/.claude/` directory exists (created by running Claude Code once)
- No existing `agents/`, `commands/`, `docs/`, `rules/`, `memory/`, or `CLAUDE.md` in `~/.claude/` (back up first if they exist)

## Setup Commands

```bash
# Define the shared config root (OneDrive path on macOS)
SHARED="/Users/jn/Library/CloudStorage/OneDrive-UniversityofExeter/Projects/.claude-shared"

# Back up existing files if any
mkdir -p ~/.claude/_backup_$(date +%Y%m%d)
for item in CLAUDE.md agents commands docs rules memory; do
  [ -e ~/.claude/$item ] && mv ~/.claude/$item ~/.claude/_backup_$(date +%Y%m%d)/
done

# Create symlinks
ln -sf "$SHARED/CLAUDE.md"  ~/.claude/CLAUDE.md
ln -sf "$SHARED/agents"     ~/.claude/agents
ln -sf "$SHARED/commands"   ~/.claude/commands
ln -sf "$SHARED/docs"       ~/.claude/docs
ln -sf "$SHARED/rules"      ~/.claude/rules
ln -sf "$SHARED/memory"     ~/.claude/memory
```

## What stays LOCAL (do NOT symlink)

- `skills/` -- Python executables, browser state, venvs differ per OS
- `projects/` -- auto-generated by Claude Code from machine-specific paths
- `cache/`, `history.jsonl`, `settings.json`, `settings.local.json` -- Claude Code internals
- `.credentials.json` -- machine-specific auth

## Verification

```bash
# Check symlinks resolve correctly
ls -la ~/.claude/CLAUDE.md  # should show -> OneDrive path
ls -la ~/.claude/agents/    # should show 3 .md files

# Verify content is accessible
cat ~/.claude/CLAUDE.md | head -5  # should show "# PhD Research Workflow"
ls ~/.claude/agents/               # should list literature-scout.md, paper-reviewer.md, experiment-tracker.md
ls ~/.claude/rules/                # should list behaviors.md, memory-flush.md, skill-triggers.md
ls ~/.claude/memory/               # should list active-tasks.json, goals.md, projects.md, today.md
```

## How sync works

- **Directories** (agents, commands, docs, rules, memory): macOS symlinks -> OneDrive folder. Edits on either machine sync via OneDrive.
- **CLAUDE.md**: On Windows uses an NTFS hard link (same volume). On macOS uses a symlink. Both point to the OneDrive copy as SSOT.
- **OneDrive** handles the actual cross-machine sync. Changes propagate within seconds when both machines are online.

## Troubleshooting

- **"Too many levels of symbolic links"**: The OneDrive folder hasn't synced yet. Wait for OneDrive to finish syncing, then retry.
- **Broken symlink**: Re-run the `ln -sf` command for the affected item.
- **Conflict files** (e.g., `CLAUDE (1).md`): OneDrive detected simultaneous edits. Manually merge and delete the conflict copy.
- **Mac OneDrive path differs**: Adjust the `SHARED` variable. Check with `ls ~/Library/CloudStorage/` to find the exact OneDrive folder name.
